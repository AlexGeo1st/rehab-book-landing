# Отчёт по учебной практике (Frontend-разработка) — Вариант 1

Тема недели: «Разработка адаптивного лендинга с интерактивными элементами на чистом HTML/CSS/JS»  
Проект: презентация книги о спортивной реабилитации баскетболистов.

---

## День 1 — Проектирование
- Выбрана тема: лендинг-презентация книги «Возвращение в игру».
- Определены секции сайта: Header, Hero, About, Features, Cases (табы), Gallery (слайдер), Contacts, Footer.
- Подготовлен контент:
  - УТП/смысл книги: управление реабилитацией по этапам и критериям прогресса
  - преимущества: диагностика, выбор методики, план нагрузки, критерии, пересмотр плана, возврат в игру
  - контакты (Москва) — в учебном формате
- Подобраны изображения (часть — предоставлена заказчиком/преподавателем, часть — заглушки не использовались).

## День 2 — Базовая вёрстка
- Создан `index.html` с семантической разметкой.
- Сделана навигация якорями по секциям.
- Добавлены базовые элементы интерфейса: кнопки, карточки, блоки кейсов.

## День 3 — Стилизация
- Подключён шрифт Google Fonts (Inter).
- Сверстана десктопная версия:
  - layout на Grid/Flexbox
  - карточки, кнопки, типографика
  - аккуратный “тёмный” стиль с акцентом
- Добавлены состояния hover/focus.

## День 4 — Адаптивность и интерактив
- Добавлены media queries для:
  - планшетов (перестройка сеток)
  - мобилки (бургер-меню, вертикальная навигация)
- Реализованы интерактивные элементы на чистом JS:
  1) Бургер-меню: открытие/закрытие, закрытие по клику на ссылку и по ESC.
  2) Табы: переключение панелей кейсов, поддержка клавиатуры (ArrowLeft/ArrowRight).
  3) Модальное окно: открытие/закрытие (overlay, кнопка, ESC), focus trap.
  4) Слайдер: массив данных (src/alt/caption), кнопки prev/next, точки, клавиатура.

## День 5 — Оптимизация и публикация
- Оптимизированы изображения: ограничение ширины, сжатие JPEG, конвертация проблемного формата в PNG.
- Подготовлена документация `README.md`.
- Подготовлены инструкции по публикации на GitHub Pages.

---

## Итог
Готовый адаптивный лендинг в открытом доступе (после публикации на GitHub Pages),
без зависимостей и без настройки сервера: достаточно открыть `index.html`.
